JavaScript-事件(二) & 事件处理程序

五月 23, 2019

事件就是用户或浏览器自身执行的某种动作。
例如 click load mouseover 等，这些都是事件的名字。
响应某个事件的函数就叫做事件处理程序（或事件侦听器）。
事件处理程序的名字以”on”开头。
因此 click 事件的事件处理程序就是 onclick，load 事件的处理程序就是 onload
为事件指定处理程序的方式有多种

事件处理程序的缺点

  1. 时差问题
    页面加载完之后，JS代码还没加载完，就单击了按钮，这样就会引发错误
    为此，很多 HTML 事件处理程序都会被封装在一个 try-catch 块中，以便错误不会浮出水面，如下:

这样，如果在上个栗子中的 showMessage() 函数没有定义之前就单击了按钮，
用户将不会看到JS错误，因为在浏览器有机会处理错误之前，错误就被捕获了

  1. 这样的扩展事件处理程序的作用域链在不同浏览器中会导致不同结果
  2. HTML与JS代码紧密耦合。如果要更换事件处理程序，就要同时改动2个地方
    这也是开发人员多数使用JS指定事件处理程序的原因

DOM0 级事件处理程序

通过JS指定事件处理程序的传统方式，就是将一个函数赋值给一个事件处理程序属性。这种为事件处理程序赋值的方法是在第四代 Web 浏览器中出现的，而且至今仍然为所有现代浏览器所支持。
原因一是简单，二是具有跨浏览器的优势
要使用JS指定事件处理程序，首先必须取得一个操作的对象的引用
每个元素（包括 window 和 document）都有自己的事件处理程序属性，这些属性通常全部小写，例如 onclick。
